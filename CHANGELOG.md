# CHANGELOG

## v0.3

### Added

- 新增 `UI/ProtectedPrograms/ProtectedProgramListView`，独立承载受保护程序列表控件。
- 新增 `UI/ProtectedPrograms/ProtectedProgramCommands`，集中处理列表相关命令。
- 新增 `UI/Main/MainContentView`，作为主内容视图后续拆分入口。
- 新增 Inno Setup 安装脚本，支持中英文安装向导和首次启动显示语言选择。
- 新增 GitHub Actions CI 与 tag 自动发布 workflow。

### Changed

- 重构主窗口 UI 结构，降低 `MainWindow.cpp` 的职责和复杂度。
- 将列表绘制、命中测试、选择状态、内部滚动和右键菜单逻辑从 `MainWindow.cpp` 拆出。
- 将项目版本号统一更新到 `v0.3` / `0.3.0`。
- 调整 Visual Studio / MSBuild 工程文件，纳入新的 UI 模块。
- 更新 `.gitignore`，忽略 Visual Studio 构建目录和常见中间产物。

### Fixed

- 修复主窗口直接维护列表状态导致的职责混杂问题。
- 修复列表右键菜单命中逻辑散落在主窗口中、不易维护的问题。
- 修复构建产物容易进入版本控制的问题。
- 修复缺少标准安装包和 release 自动化的问题。

## v0.2

### Added

- 新增受保护程序列表，用于保存需要 FFKeyLock 接管输入法保护的程序。
- 新增添加当前前台程序功能。
- 新增通过文件选择器添加 `.exe` 的功能。
- 新增浏览受保护程序列表窗口，可查看程序名称和已记录路径。
- 新增列表右键菜单，可复制程序名或打开程序所在目录。
- 新增中文 / 英文界面切换和主题设置。

### Changed

- 改进程序列表的配置保存逻辑，删除或新增条目后会同步更新配置。
- 扩展设置、通知和帮助菜单，集中放置更多常用操作。
- 优化受保护程序相关按钮和列表区域的主窗口布局。

### Fixed

- 修复只保存程序名时缺少路径提示的问题。
- 修复删除选中程序后列表选择状态可能不稳定的问题。
- 修复程序列表为空时缺少清晰提示的问题。

## v0.1

### Added

- 初始版本：提供轻量级 Win32 托盘工具基础框架。
- 新增前台窗口检测和受保护程序识别能力。
- 新增游戏保护模式，可在受保护程序激活时切换到英文输入法。
- 新增离开受保护程序后恢复之前输入法状态的能力。
- 新增保护模式、自动检测、开机启动等基础开关。
- 新增 Windows 键锁定功能，减少游戏中误触打断。
- 新增 INI 配置文件，用于保存用户设置。

### Changed

- 使用原生 Win32、C++20 和 Windows Shell 托盘 API 构建轻量桌面应用。
- 使用短间隔前台窗口轮询完成自动检测，避免引入额外后台服务。
- 将用户配置保存到 `%APPDATA%\FFKeyLock\config.ini`。

### Fixed

- 修复游戏过程中输入法误切换造成操作不可靠的问题。
- 修复离开游戏后输入法状态无法自动恢复的问题。
- 修复误触 Windows 键可能打断游戏的问题。
